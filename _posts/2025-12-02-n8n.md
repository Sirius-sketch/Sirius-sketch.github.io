## 构建环境
1.创建一个n8n文件夹，包含Dockerfile和docker-compose.yml

```dockerfile
FROM n8nio/n8n:latest

USER root

# 1. 使用 apk 安装 python3 和 pip
# build-base 和 python3-dev 是为了防止 PyMuPDF 编译时缺依赖
RUN apk add --update --no-cache python3 py3-pip build-base python3-dev

# 2. 安装 Python 库
# --break-system-packages 是必须的，因为新版 Python 保护机制限制直接 pip install
RUN pip3 install PyMuPDF pypdf --break-system-packages

USER node
```


```yaml
version: '3'

services:
  n8n:
    build: .
    image: n8n-read-python
    ports:
      - "5678:5678"
    environment:
      - N8N_HOST=localhost
      - N8N_PORT=5678
      - N8N_PROTOCOL=http
      - GENERIC_TIMEZONE=Asia/Shanghai
    volumes:
      - ~/.n8n:/home/node/.n8n
```


2.构建镜像：docker compose build  
启动：n8ndocker compose up -d（之后可直接在docker desktop里面启动）  
在浏览器打开http://localhost:5678  

3.在Credentials中添加API key，我用的moonshot（base URL填https://api.moonshot.cn/v1/
）和github API（获取：GitHub Settings -> Developer settings -> Personal access tokens -> Generate new token (Classic)）

### 本地版n8n（为了直接打开vscode，把docker的迁移到本地的）

启动：n8n start

```bash
# 1. 设置代码存放目录
# 不要放在 /mnt/c/... (Windows盘符)，因为跨文件系统 Git 速度很慢
cd /home/你的用户名/projects

# 2. 从上一节点获取 URL (假设上一节点叫 HTTP Request)
REPO_URL="{{ $json.content }}"

# 3. 提取仓库名
FOLDER_NAME=$(basename "$REPO_URL" .git)

# 4. 克隆代码 (如果不存在)
if [ ! -d "$FOLDER_NAME" ]; then
  git clone "$REPO_URL"
fi

# 5. 用 VS Code 打开该文件夹
# WSL 里的 code 命令会神奇地呼叫 Windows 端的 VS Code 并远程连接到这里
code "$FOLDER_NAME"
```


## 搭建工作流
报错Cannot find module 'fs'，这是n8n安全机制默认把fs文件系统操作锁住，  
1.在终端输入：nano ~/.bashrc；2.粘贴：export NODE_FUNCTION_ALLOW_BUILTIN=fs,path；3.输入 source ~/.bashrc 让它立刻生效。
